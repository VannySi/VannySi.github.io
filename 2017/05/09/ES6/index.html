let 与 var

  • 同一个作用域里，如果命名重复的话，var 后者会覆盖掉前者，而 let 会报错
  • var 只有全局作用域和函数作用域，let 相较于 var 增加了块作用域
  • let 解决了 var 的变量泄露问题，var 一般使用闭包来解决这个问题

const

  • 同一个作用域里，如果命名重复的话会报错，不同作用域里会不影响
  • const 定义常量，值不能改变
  • const 允许变量赋值
  • const 的实际用途

for-in 与 for-of

for-in是为普通对象设计的，适用于遍历得到字符串类型或者对象的键，而不适用于数组遍历。
在使用for-in遍历数组时会出现以下问题：

  1. for(let k in arr) 中k的值不是数字而是字符串”0”、”1”、”2”
  2. for-in循环体除了遍历数组元素外，还会遍历自定义属性，甚至连数组原型链上的属性都能被访问到
  3. 在某些情况下，for-in可能按照随机顺序遍历数组元素

由于在使用for-in遍历数组会出现一些问题，所以ES6增加了for-of： 语法简洁，修复for-in的问题，可以使用break、continue、return关键字(相比于forEach)

  • 支持遍历 DOM NodeList(for-in遍历得到的并不是子节点)
  • 支持字符串的遍历，会把字符串作为一组 Unicode 的字符进行遍历(for-in也可以做到)

  • 还支持应用于 Map 和 Set 对象(Map 和 Set 对象还没研究，暂搁)

  • 注意： for-of并不适用于处理原有的原生对象(包括JSON对象)，处理原有的原生对象时可以使用for-in或者内置的Object.keys()
